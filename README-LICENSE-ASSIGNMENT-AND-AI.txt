To confirm to the expectations of the ACM's research policy about publications,
we are keeping notes about the use of Large Language Models, as noted here.

LLM's have been used in the construction of this software in only the following ways:

* As learning tools to self-teach Racket and Racket libraries, e.g., "semantic-search" of racket docs, example-generation
  + I (Dave W) copy-pasted code for the hex-str function that formats base 16
* As tools to identify problems without actually changing or adding code
* As a helper to give ideas on how to make the visualizer better
* To suggest a name from Greek mythology for seeing HERA more clearly (thus, "Iris")
* To tell existing Racket in-built functions so don't have to create them ourselves
* To suggest next steps of what to do in the GUI code to connect it with the backend API
